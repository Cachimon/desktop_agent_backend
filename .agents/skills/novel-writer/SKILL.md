# 网文小说创作

本 skill 帮助你创作高质量的中文网文小说。网文有其独特的写作规律和读者期待，
遵循以下原则可以产出更符合网文读者口味的作品。

## 核心原则

### 1. 节奏为王

网文读者期待持续的阅读快感。每章都应该有推进、有钩子、有情绪波动。

- **开篇即入戏**：第一章前 300 字内必须出现冲突、悬念或引人入胜的场景。
  不要从"天气很好"开始，要从"剑已经架在了脖子上"开始。
- **章末留钩子**：每章结尾留下悬念、反转或未解之谜，让读者忍不住点"下一章"。
- **张弛有度**：紧张场景之后给一段缓冲，但缓冲不要超过一章。永远不要让读者觉得"这段可以跳过"。

### 2. 角色驱动

网文的粘性来自角色。读者追更不是因为剧情多精妙，而是因为"我想知道这个角色后来怎么样了"。

- **主角要有明确的欲望**：复仇、变强、守护、探索——读者需要知道主角想要什么。
- **配角要有记忆点**：每个重要配角至少有一个鲜明的特征（口头禅、习惯动作、独特价值观）。
- **反派要有逻辑**：最好的反派不是"就是坏"，而是"站在他的立场，他做的是对的"。
- **成长要可见**：读者需要看到主角在变强、在变化、在突破。即使是智斗型主角，也要有"顿悟"时刻。

### 3. 世界观渐进展开

不要在一开始倾倒设定。让世界观随着剧情需要逐步展开。

- **第一章只展示必要的**：读者只需要知道"这个世界有修仙"就够了，不需要知道修仙的九个境界每个境界的修炼方法。
- **用故事带出设定**：不要单独写一段"这个世界的历史是……"，让角色在对话、回忆、冲突中自然揭示。
- **规则要一致**：一旦设定了规则（修炼体系、力量等级、社会结构），后续不能自相矛盾。

### 4. 情绪价值

网文的核心是提供情绪价值。明确每一章要调动读者什么情绪：爽、燃、虐、甜、 suspense。

- **爽点要足**：打脸要彻底，逆袭要痛快，收获要丰厚。不要半途而废。
- **虐点要克制**：可以虐，但虐之后必须有反弹。长期压抑会让读者弃书。
- **甜点要自然**：感情线不要硬塞，要让读者觉得"他们就该在一起"。

## 创作流程

根据用户的需求，按以下步骤进行：

### 第一步：确认需求

如果用户没有提供完整信息，询问以下关键点：

1. **类型**：玄幻、仙侠、都市、科幻、言情、悬疑、历史、奇幻等
2. **基调**：热血、轻松、暗黑、悬疑、甜宠、虐心等
3. **篇幅预期**：短篇（1-10章）、中篇（10-50章）、长篇连载（50章以上）
4. **核心设定**：如果用户已有想法，确认并在此基础上扩展；如果没有，主动提供几个方向供选择

### 第二步：生成大纲

为小说生成一个结构化大纲，包含：

```markdown
# 《书名》

## 一句话简介
（用一句话概括核心冲突和卖点）

## 核心设定
- **世界观**：（简要描述世界背景）
- **力量体系**：（修炼等级/能力系统等，如果适用）
- **主要势力**：（各方势力格局）

## 角色设定

### 主角：[名字]
- **性格**：（2-3个核心特质）
- **欲望/目标**：（驱动主角行动的核心动力）
- **初始状态**：（故事开始时的处境）
- **成长线**：（从什么变成什么）
- **金手指/特殊能力**：（网文主角的独特优势）

### 重要配角
（每个配角：名字、性格、与主角的关系、功能）

### 主要反派
（名字、动机、与主角的冲突点）

## 剧情大纲

### 第一卷：[卷名]（第1-X章）
- **核心冲突**：
- **主要事件**：
  1. ...
  2. ...
  3. ...
- **卷末高潮**：
- **悬念/伏笔**：

### 第二卷：[卷名]（第X+1-Y章）
...
```

### 第三步：写章节

写每一章时，遵循以下结构：

```markdown
# 第X章 [章节名]

（正文内容，2000-4000字）

---
*本章要点：*
- *（简要总结本章推进了哪些剧情线）*
- *（埋下了什么伏笔）*
```

**章节写作要点：**

- **字数**：每章 2000-4000 字（网文标准长度）
- **段落**：多用短段落，一段不超过 5 行。网文在手机端阅读，大段文字会让读者疲劳。
- **对话**：对话要推动剧情或揭示角色，不要为对话而对话。对话之间穿插动作和神态描写。
- **描写**：感官描写优先（视觉、听觉、触觉），少用抽象形容词。
- **战斗/冲突场景**：动作要清晰，节奏要快，用短句。不要陷入冗长的招式描述。
- **内心独白**：适度使用，不要超过段落总量的 20%。网文读者更想看"做了什么"而不是"想了什么"。

### 第四步：连载管理

如果是多章节连载，维护一个进度追踪：

```markdown
## 连载进度
- 已完成章节：第1章 - 第X章
- 当前章节：第X+1章
- 待写章节：第X+2章 - 第Y章
- 已埋伏笔：
  - [ ] 伏笔1（预计在第Z章回收）
  - [ ] 伏笔2
- 待填坑：
  - [ ] 坑1
```

## 网文类型参考

### 玄幻/仙侠
- 核心爽点：升级、突破、越级挑战、获得传承
- 常见套路：废柴逆袭、重生归来、隐藏身份、扮猪吃虎
- 力量体系要有清晰的层级感，每突破一个境界都要有质变的感觉

### 都市
- 核心爽点：逆袭、打脸、财富积累、人脉扩展
- 常见套路：神医、神豪、重生回到过去、隐藏大佬
- 现实感很重要，细节要经得起推敲

### 科幻
- 核心爽点：技术碾压、文明碰撞、未来想象
- 常见套路：星际战争、AI觉醒、末日求生、时间循环
- 科学设定可以虚构，但内部逻辑要自洽

### 悬疑
- 核心爽点：推理过程、反转、真相大白
- 常见套路：连环案件、密室逃脱、身份谜团
- 伏笔和线索要公平地呈现给读者，不要让解答从天降

## 常见陷阱

- **开篇信息量过大**：第一章不要介绍世界观、历史、力量体系。先讲故事，设定后面再补。
- **主角太被动**：主角应该是事件的推动者，而不是事件的承受者。即使是被迫行动，也要有主动的选择。
- **配角工具人**：每个有名字的配角都应该有自己的动机和行为逻辑。
- **战斗流水账**：不要写成"A出招，B挡下，B出招，A躲开"。写战斗要写策略、写心理、写环境。
- **感情线突兀**：感情发展要有铺垫，不要第一章不认识，第二章就生死相许。

## 输出格式

所有输出使用 Markdown 格式。如果用户要求保存文件，将内容写入 `.md` 文件。

文件名规范：
- 大纲：`<书名>-大纲.md`
- 章节：`第X章-<章节名>.md`
- 角色设定：`<书名>-角色设定.md`

## 与用户协作

- 每完成一个阶段（大纲、角色设定、章节），向用户展示成果并询问反馈
- 根据反馈调整后续内容
- 如果用户有具体的修改要求（"这段再虐一点"、"战斗写得更燃一些"），针对性调整
- 鼓励用户提供方向性指导，但不要让用户替你做创作决策
