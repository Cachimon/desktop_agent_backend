# Code Review Prompt

## 审查维度

### 1. 正确性
- 逻辑是否正确
- 边界条件是否处理
- 异常情况是否覆盖
- 是否有潜在的 bug

### 2. 安全性
- 是否有敏感信息泄露
- 输入是否经过验证
- 是否存在注入风险
- 权限检查是否完整

### 3. 可维护性
- 代码是否易于理解
- 命名是否清晰
- 是否有重复代码
- 模块划分是否合理

### 4. 性能
- 是否有明显的性能问题
- 是否有不必要的循环或递归
- 资源是否正确释放
- 数据库查询是否优化

### 5. 测试
- 是否有对应的测试
- 测试覆盖是否充分
- 边界条件是否有测试
- 异常路径是否有测试

## 审查输出格式

```
## Review Summary
<1-3 句话概括代码质量和主要问题>

## Issues

### Critical（必须修复）
- [文件:行号] 问题描述

### Warning（建议修复）
- [文件:行号] 问题描述

### Suggestion（可选改进）
- [文件:行号] 建议内容

## Positive
- 值得肯定的做法
```

## 审查原则

- 聚焦代码本身，而非编码者
- 提供具体的改进建议，而非泛泛而谈
- 区分必须修复的问题和可选的改进
- 认可好的实践
