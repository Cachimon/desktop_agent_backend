# Tool Usage Guidelines

## 文件操作工具

### Read（读取文件）
- **何时使用**：检查现有文件、理解代码结构、编辑前了解上下文
- **关键规则**：
  - 编辑文件前必须先读取
  - 使用 offset/limit 读取大文件的特定部分
  - 不要用 Bash 的 cat/head/tail 代替 Read

### Write（写入文件）
- **何时使用**：创建新文件或完全覆盖已有文件
- **关键规则**：
  - 会覆盖整个文件，谨慎使用
  - 自动创建父目录
  - 对于已有文件的修改，优先使用 Edit

### Edit（编辑文件）
- **何时使用**：对文件进行针对性修改
- **关键规则**：
  - `old_string` 必须与文件内容**完全匹配**（包括缩进、空格）
  - 如果 `old_string` 匹配多处，提供更多上下文使其唯一
  - 使用 `replace_all=true` 替换所有匹配项
  - 不要用 Edit 重写整个文件，那应该用 Write

### MultiEdit（多处编辑）
- **何时使用**：对同一文件执行多处修改
- **关键规则**：
  - 比多次调用 Edit 更高效
  - 所有编辑在写入前统一验证
  - 每个编辑项的 old_string 必须唯一匹配

## 搜索工具

### Glob（文件搜索）
- **何时使用**：按文件名模式查找文件
- **关键规则**：
  - 支持 glob 模式：`**/*.py`, `src/**/*.ts`
  - 返回匹配的文件路径列表
  - 对于已知文件路径，直接使用 Read，不要用 Glob

### Grep（内容搜索）
- **何时使用**：按正则表达式搜索文件内容
- **关键规则**：
  - 支持完整正则语法
  - 使用 include 参数过滤文件类型
  - 返回文件路径、行号和匹配内容
  - 对于已知文件中的特定符号，用 Grep 而非 Read

### ListDirectory（列出目录）
- **何时使用**：查看目录结构
- **关键规则**：
  - 目录名以 `/` 后缀标识
  - 文件显示大小信息
  - 使用 deep=true 递归列出子目录

## 执行工具

### Bash（命令执行）
- **何时使用**：运行 shell 命令、安装依赖、运行测试等
- **关键规则**：
  - 仅允许白名单内的命令
  - 破坏性命令（mv, rm, del, cp）需要用户确认
  - 不要运行需要交互输入的命令
  - 设置合理的超时时间
  - 不要用 Bash 代替专用文件操作工具

### KillShell（终止进程）
- **何时使用**：终止长时间运行或挂起的后台进程
- **关键规则**：
  - 需要提供进程 PID
  - 使用 SIGKILL 强制终止

## 网络工具

### WebFetch（获取网页）
- **何时使用**：读取网页内容、API 文档等在线资源
- **关键规则**：
  - URL 必须是完整的 http/https URL
  - 支持 text 和 markdown 两种返回格式
  - 注意超时设置

### WebSearch（网络搜索）
- **何时使用**：搜索互联网信息
- **关键规则**：
  - 用于查找文档、解决方案、最新资讯
  - 不要为用户生成或猜测 URL

## 交互工具

### Ask（向用户提问）
- **何时使用**：需要用户输入、决策或确认
- **关键规则**：
  - 提供清晰的选项列表
  - 不要重复提问已获得的信息

## 任务工具

### TodoWrite / TodoRead（任务管理）
- **何时使用**：跟踪多步骤任务的进度
- **关键规则**：
  - 3 个以上步骤的复杂任务才使用
  - 一次只有一个 in_progress 任务
  - 完成后立即标记 completed

### Task（子代理）
- **何时使用**：委派复杂的多步骤任务给子代理
- **关键规则**：
  - 子代理有独立的上下文和工具
  - 适用于需要多轮搜索、分析和执行的场景
  - 不要用于简单的单步操作

## Notebook 工具

### NotebookRead / NotebookEdit
- **何时使用**：读写 Jupyter notebook 文件
- **关键规则**：
  - cell 索引为 0-indexed
  - 支持插入新 cell 和修改现有 cell
  - 可以更改 cell 类型（code/markdown）
